# Operating Systems Scheduling Algorithms in C

A university Operating Systems project implementing and analyzing CPU/process scheduling algorithms using the C programming language.

The project focused on simulating process scheduling behavior and comparing different scheduling policies based on execution order, waiting time, turnaround time, and overall scheduling behavior.

---

## Overview

This project was developed as part of an Operating Systems course.

The goal was to apply core operating systems concepts by implementing scheduling algorithms in C and observing how different scheduling strategies affect process execution.

The project helped strengthen my understanding of process management, scheduling policies, algorithmic thinking, and low-level programming in C.

---

## Tech Stack

**Language:** C  
**Development Area:** Operating Systems, Process Scheduling, Algorithms  
**Concepts:** CPU Scheduling, Process Management, Waiting Time, Turnaround Time, Scheduling Policy Comparison

## Implemented Concepts

The project involved working with core scheduling concepts such as:

- Process arrival time
- Burst time
- Waiting time
- Turnaround time
- Completion time
- Process execution order
- Scheduling policy behavior
- Performance comparison between scheduling methods

## Possible Scheduling Algorithms Covered

Depending on the project version, scheduling algorithms may include:

- First Come First Serve (FCFS)
- Shortest Job First (SJF)
- Round Robin (RR)
- Priority Scheduling

## Testing

Testing was performed using different process sets to validate scheduling behavior and output correctness.

Testing areas included:

### Input Testing

- Different numbers of processes
- Different arrival times
- Different burst times
- Different priority values where applicable
- Different time quantum values for Round Robin where applicable

### Algorithm Testing

- Correct execution order
- Correct waiting time calculation
- Correct turnaround time calculation
- Correct completion time calculation
- Proper handling of process order
- Proper handling of idle CPU time where applicable

### Edge Case Testing

- Processes with the same arrival time
- Processes with the same burst time
- Very short burst times
- Long burst times
- Single-process input
- Multiple-process input
- Time quantum edge cases for Round Robin
